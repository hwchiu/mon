What We Validate Here
  • Zone CIDR lookup on real Azure-routed packets (node IPs)
  • Both egress AND ingress checks on forward + return traffic
  • AKS (Azure CNI) + bare-metal VM (Podman) agent paths side-by-side
  • Wall-clock propagation measurement across real network
  • Controller outage → agents keep last-good policy
  • Overlay vs traditional CNI impact on visible IPs
  • tc attach on real Azure NICs (including accelerated networking)

Core Scenarios
  1. Ground matrix applied → traffic from Internal to DMZ blocked on both sides
  2. ZoneRule added → both agents receive update, ringbuf shows new verdict, traffic flows
  3. Full TCP session (forward + return) only succeeds when bidirectional consent exists
  4. Measure time from kubectl apply to first successful packet on both agents
  5. Scale controller to 0 → traffic continues with last-good policy
