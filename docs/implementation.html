10-Phase Roadmap

Implementation Plan

Incremental, independently valuable phases. Strict TDD. Progressive e2e from unit tests to real Azure cross-VNet traffic.

The plan delivers a working DFW system through 10 ordered, mergeable phases. Each phase produces code that builds, tests green, and delivers standalone value.

1-4Foundations
  • 1. Bootstrap — go.mod, Makefile (bpf targets + LLVM agent image), cmd/ stubs, basic CI
  • 2. CRDs & Types — Zone, GroundRule, ZoneRule, PolicyVersion with validation + samples matching the 4-zone matrix
  • 3. Policy Engine — ground + zone rule merge (OR logic, ground always wins), conflict detection, compilation to the four eBPF map structures + immutable version
  • 4. Controller — controller-runtime skeleton, leader election, reconciliation that drives the engine and creates PolicyVersion CRs
5-7Data Plane & Distribution
  • 5. eBPF Program — data-driven C (zone LPM lookup on host interfaces, dual egress+ingress verdict check, tc-BPF primary, ringbuf audit, double-buffer maps, early same-zone bypass)
  • 6. Agent — runtime LLVM compile of the shipped C at boot, cilium/ebpf loader + tc attach (only admin-specified ifaces), atomic map apply, last-known-good recovery, health/metrics
  • 7. gRPC Channel — signed policy updates (map data blobs), agent registration, unidirectional push, basic delta + backoff, status/audit feedback
8-10Production & Validation
  • 8. Safety & Security — strict atomic flips, restart recovery, signature verification (fail-closed), version skew tracking, breakglass support
  • 9. Packaging & Coexistence — Helm for controller + agent DaemonSet, Podman/systemd examples for VMs, explicit tc priority + CNI coexistence tests in kind + bare metal
  • 10. Observability & Ops — Prometheus metrics, ringbuf → structured audit, simulation/dry-run endpoint, runbooks, full propagation SLA validation
Core Principles
  • Every phase builds & tests green independently
  • Progressive environments: unit → envtest → kind → real Azure multi-VNet
  • The Azure test environment is the primary validation vehicle for the dual-consent model
  • YAGNI: focus on the zone-centric ground + zone rules model first
